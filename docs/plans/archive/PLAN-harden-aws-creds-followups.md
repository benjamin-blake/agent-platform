## Constraints
- Per AGENTS.md and Decision 35: `terraform apply` is human-gated; present `plan` output before any apply. IAM apply precedes any Lambda deploy (none in scope).
- Decision 60 (no silent fallbacks): the degraded-mode credential path and the `read_priority_queue` cache fallback MUST emit loud, actionable guidance (visible staleness/empty-cache note); they must never silently weaken a gate. The genuine-infra-fault case (creds OK but query `None`) still hard-fails loudly.
- Decision 55 (no workaround loops): the preserved `on_failure = continue` on the provisioners is pre-existing idempotency behaviour (`CREATE TABLE IF NOT EXISTS` returns `FAILED` when the object already exists), NOT a new silent-failure workaround. Justify this in a code comment.
- Warehouse-as-source-of-truth: the `read_priority_queue` fallback only READS the `.priority-queue.jsonl` read-cache; it never restages or writes from a local file (no resurrection loop). The portal Decision write (VP 11) goes through `ops_data_portal`, never a direct file write.
- Memory policy: durable learnings are proposed as CLAUDE.md edits, not auto-memory.
- No emojis; ASCII hyphens only; type hints + `async` for I/O; ruff line length 127; `bin/venv-python` not `python`.
- Out of scope (do NOT touch): production-code default literals (`setup.py`, `scripts/build_lambda.py`, `scripts/run_scheduled_agent.py`, `scripts/migrate_ops_data.py` source-profile, `src/common/*` docstrings); passive narrative docs (`docs/ARCHITECTURE*.md`, `README.md`, `config/README.md`, `terraform/README.md`); the dead `.github/` non-workflow files (`copilot-instructions.md`, `prompts/*.prompt.md`) - per human direction these are not used again; append-only ledgers (`docs/CHANGELOG.md`, `docs/DECISIONS.md` history); removal of the redundant `AgentPlatformRuntime` inline policy (separate breakglass follow-up).
