## Constraints
- **Strangler Fig only**: Area A extractions must use the re-export pattern (move functions to new module, import back in the original). No callers should break. No @patch() paths should break until a dedicated follow-up migration PR.
- **Gating recs**: Area A decomposition of executor files must not begin until these open recs that actively modify the same files are closed or superseded: rec-413 (Critical, execute_recommendation.py), rec-423 (Critical, postflight.py), rec-404 (High, execute_recommendation.py), rec-370 (High, executor preflight). The /implement session must declare these as dependencies on every child rec.
- **Acknowledged conflict recs (not gated)**: rec-424 (M, track failed-attempt residue, execute_recommendation.py) and rec-426 (S, preflight acceptance challenge, execute_recommendation.py) both modify the same file as Area A extractions. They are not gated because their scope is additive (new functions) rather than restructuring existing ones, but the /implement session should evaluate whether they should be bundled or sequenced. rec-381 (L, model routing wiring) and rec-334 (L, compound branch isolation) also touch extraction targets but their L effort makes near-term execution unlikely.
- **Test namespace migration**: Area A re-exports MUST preserve existing `@patch()` resolution. No `@patch()` path changes in this plan; a dedicated follow-up migration PR handles path updates after re-exports are stable.
- **No Docker**: Lambda packaging via zip/S3 as per copilot-instructions.md.
- **YAML config in `config/`**: All governance configuration (agent_governance.yaml, agent_tool_tiers.yaml) lives in config/, not scattered in frontmatter or ad-hoc locations.
- **Windows Git Bash**: All automation scripts must be Python, not shell. subprocess calls must use `encoding='utf-8', errors='replace'`.
- **Waiver mechanism**: Hard gates (SLOC, complexity, token budget, tool tier) must all support explicit waivers with decision-id references, so legitimate orchestrators are not blocked.

## Context
- **Decision 42** (Three-Tier Workflow Architecture): Defines the /plan -> /implement -> /develop-executor separation of concerns. This plan adds governance gates that enforce those boundaries structurally.
- **Decision 40** (Copilot SDK Migration -- Deferred): The CLI subprocess approach is the current reality. Tool tier standardisation (Area D) must work with `copilot_call()` as-is.
- **Decision 41** (Scalable Feature Architecture): The three-layer data pipeline is a future growth vector. Growth governance (this plan) ensures that when Phase 1.5/2 feature recs are implemented, the resulting code stays within structural limits.
- **Known Gotcha -- Monolith-to-package refactor: Test namespace migration**: All `@patch("module.symbol")` calls must be updated to new submodule locations after extraction. This is a two-phase operation (extract+re-export, then migrate patches in a separate PR).
- **Known Gotcha -- ruff format duplicate import consolidation**: When adding re-exports, use one consolidated import block to avoid ruff silently dropping symbols.
- **Complexity warnings**: `execute_recommendation.py` (complexity 32, threshold 29.29), `validate.py` (complexity 39, threshold 29.29) are already flagged.
- **rec-curator current state**: Exists as a scheduled agent in schedule.yaml (Monday 08:00 UTC, claude-opus-4.6). Runs via generic hourly Lambda dispatch. Has no dedicated EventBridge rule. Output is findings JSONL, not priority queue format. Needs rewrite for queue output + dedicated infra.
- **142 open recs**: The priority queue (Area E) is essential to manage this backlog. Without automated prioritisation, human session prep time is the bottleneck.
